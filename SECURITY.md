# Security Policy

## Reporting Security Issues

Do not open public GitHub issues for security vulnerabilities. Instead, please report security vulnerabilities via email to the maintainer.

When reporting a vulnerability, include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

We will work with you to understand and address the issue.

## Supported Versions

Security updates are provided for the latest minor version of strict-module. Update regularly to receive security patches.
